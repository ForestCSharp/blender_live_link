GI Probe Octree Placement Explained

Probe placement is now built from a sparse, pruned octree instead of a fixed dense grid. The CPU builds the layout from visible mesh bounds, shares probes across octree cell corners, uploads the compact structure to the GPU, and the lighting shader walks the octree to find the best cell for each shaded point.

CPU layout Sparse octree GPU lookup

The important change is that probe density now follows scene occupancy. Child octree nodes are only created where their bounds intersect visible mesh bounds. Missing child slots are pruned, and shader traversal falls back to the deepest valid ancestor payload. 

if (!gi_scene_bounds_intersect_any(child_bounds, in_geometry_bounds))
{
    continue;
}

child_indices[child_slot] = gi_scene_build_sparse_octree_node(...);

Shared Data Model

The CPU and shaders share the GI structs from gi_helpers.h. Static asserts in render/gi.h keep the C++ storage layout aligned with shader storage buffers. 

struct GI_Cell
{
    int probe_indices[8];
};

struct GI_Probe
{
    vec4 position;
    int atlas_idx;
    float max_radial_depth;
    int padding[2];
};

struct GI_OctreeNode
{
    vec4 min;
    vec4 max;
    int is_leaf;
    int child_indices[8];
    int payload_index;
    int padding[2];
};
Buffer Contains Used By
octree_nodes Bounds, child links, leaf flag, and payload index for each octree node. lighting.glsl octree traversal.
cells Eight probe indices for the trilinear cell attached to an octree node. lighting.glsl probe interpolation.
probes Probe position, atlas assignment, and maximum radial depth. Probe capture, debug draw, and lighting.
sh9_coefficients / sg9_lobes Radiance data sized from the final probe count. Radiance projection and lighting modes.

Scene Bounds

Layout rebuild starts by collecting bounds for objects that are both visible and mesh-backed. Light-only objects, invisible objects, and non-mesh objects do not place probes. 

for (auto const& [unique_id, object] : in_state.scene.objects)
{
    if (!object.visibility || !object.has_mesh)
    {
        continue;
    }

    const BoundingBox object_bounds = object_get_bounding_box(object);
    out_geometry_bounds.add(object_bounds);
    out_scene_bounds.min = HMM_MinV3(out_scene_bounds.min, object_bounds.min);
    out_scene_bounds.max = HMM_MaxV3(out_scene_bounds.max, object_bounds.max);
}

The combined bounds are expanded to a cube so octree subdivision is uniform in X, Y, and Z. If no visible mesh exists, the scene uses a fallback cube centered at the origin and skips sparse node payload generation. 

out_scene_bounds = gi_scene_expand_bounds_to_cube(out_scene_bounds);

Depth And Lattice

GI Octree Depth controls the maximum subdivision depth. The current range is 1..4, and max-depth lattice divisions are 1 << depth. Probe positions are placed on that max-depth lattice even when the cell payload belongs to a coarser ancestor. 

i32 gi_scene_leaf_divisions_from_depth(const i32 in_octree_depth)
{
    return 1 << in_octree_depth;
}

HMM_Vec3 gi_scene_lattice_position(
    const BoundingBox& in_bounds,
    const i32 in_leaf_divisions,
    const GI_Coords in_coords)
{
    const f32 cell_extent = (in_bounds.max.X - in_bounds.min.X) / (f32)in_leaf_divisions;
    return in_bounds.min + HMM_V3(
        (f32)in_coords.x * cell_extent,
        (f32)in_coords.y * cell_extent,
        (f32)in_coords.z * cell_extent
    );
}
Why use one lattice: using max-depth coordinates lets probes be shared between neighboring cells, even when those cells live at different octree depths.

Cell Payloads

Every octree node that is actually created receives a GI_Cell payload. The payload stores eight corner probes. Coarser nodes use a larger coordinate scale so their corners still line up with the max-depth lattice. 

const i32 coord_scale = 1 << (out_gi_scene.octree_depth - in_depth);
const f32 cell_extent = gi_scene_cell_extent_from_depth(out_gi_scene, in_depth);
const f32 required_radial_depth = gi_scene_max_radial_depth_from_cell_extent(cell_extent);

for (i32 corner_z = 0; corner_z < 2; ++corner_z)
{
    for (i32 corner_y = 0; corner_y < 2; ++corner_y)
    {
        for (i32 corner_x = 0; corner_x < 2; ++corner_x)
        {
            const GI_Coords probe_coords = {
                (in_coords.x + corner_x) * coord_scale,
                (in_coords.y + corner_y) * coord_scale,
                (in_coords.z + corner_z) * coord_scale,
            };
            cell.probe_indices[corner_index] =
                gi_scene_get_or_create_probe(out_gi_scene, probe_indices_by_corner, probe_coords, required_radial_depth);
        }
    }
}

Probe corner coordinates are hashed into a 64-bit key. If another cell already created the same corner probe, the existing probe is reused and its max_radial_depth is raised if the new cell needs a larger capture radius.

Pruned Children

A node subdivides only into child octants whose bounds intersect at least one visible mesh bounding box. This is broad-phase pruning against object bounds, not triangle-accurate occupancy, which keeps layout rebuilds cheap. 

for (i32 child_z = 0; child_z < 2; ++child_z)
{
    for (i32 child_y = 0; child_y < 2; ++child_y)
    {
        for (i32 child_x = 0; child_x < 2; ++child_x)
        {
            const BoundingBox child_bounds = gi_scene_child_bounds(in_bounds, child_x, child_y, child_z);
            if (!gi_scene_bounds_intersect_any(child_bounds, in_geometry_bounds))
            {
                continue;
            }

            const i32 child_slot = child_x + child_y * 2 + child_z * 4;
            const i32 child_index = gi_scene_build_sparse_octree_node(...);
            octree_nodes[node_index].child_indices[child_slot] = child_index;
        }
    }
}

If no children are created, the node remains a leaf. If some children are absent, those slots stay -1. The shader can still shade points in that region by using the node's own payload.

Fallback Probe

After building the sparse octree, the scene appends one fallback probe at the center of the scene bounds. Its index is stored separately from the non-fallback probe count. 

out_gi_scene.non_fallback_probe_count = (i32)out_gi_scene.probes.length();
out_gi_scene.fallback_probe_index = out_gi_scene.non_fallback_probe_count;

out_gi_scene.probes.add(
    gi_scene_make_probe(gi_scene_bounds_center(out_gi_scene.scene_bounds), fallback_radial_depth)
);

Lighting uses this fallback when a shaded point is outside the octree root or when no payload can be found. Debug probe rendering only draws non_fallback_probe_count, so the fallback probe is not drawn as a normal layout probe.

GPU Traversal

lighting.glsl receives gi_octree_node_count and the GIOctreeNodesBuffer. It starts at node 0, checks that the surface position is inside the root bounds, and then descends toward the child slot containing the position. 

int find_gi_octree_payload_index(vec3 position)
{
    if (gi_octree_node_count <= 0)
    {
        return -1;
    }

    int node_index = 0;
    GI_OctreeNode node = gi_octree_nodes[node_index];
    if (!gi_octree_is_valid_position(node, position))
    {
        return -1;
    }

    int payload_index = node.payload_index;
    for (int i = 0; i < GI_MAX_OCTREE_SEARCH_DEPTH; ++i)
    {
        if (node.is_leaf != 0)
        {
            return payload_index;
        }

        int child_slot = gi_octree_child_slot(node, position);
        int child_index = node.child_indices[child_slot];
        if (child_index < 0 || child_index >= gi_octree_node_count)
        {
            return payload_index;
        }

        node = gi_octree_nodes[child_index];
        if (node.payload_index >= 0)
        {
            payload_index = node.payload_index;
        }
    }

    return payload_index;
}

Returning the last valid payload is what makes pruning work. A shaded point can land in a missing child slot and still use the ancestor cell instead of failing the GI lookup.

Probe Blending

Once the shader has a payload index, it loads the GI_Cell, computes local alpha from corner probes 0 and 7, and trilinearly blends the eight probe samples. Existing normal-facing, occlusion, and weight-threshold logic still apply after the octree lookup. 

GI_Cell cell = gi_cells[cell_index];

vec3 cell_min_pos = gi_probes[cell.probe_indices[0]].position.xyz;
vec3 cell_max_pos = gi_probes[cell.probe_indices[7]].position.xyz;
vec3 cell_extent = max(cell_max_pos - cell_min_pos, vec3(0.00001));
vec3 alpha = clamp((position - cell_min_pos) / cell_extent, 0.0, 1.0);

for (int i = 0; i < 8; ++i)
{
    int probe_index = cell.probe_indices[i];
    if (probe_index == -1) { continue; }

    float x_side = float(i & 1);
    float y_side = float((i >> 1) & 1);
    float z_side = float((i >> 2) & 1);

    float trilinear_weight =
        (x_side > 0.5 ? alpha.x : 1.0 - alpha.x) *
        (y_side > 0.5 ? alpha.y : 1.0 - alpha.y) *
        (z_side > 0.5 ? alpha.z : 1.0 - alpha.z);
}

Rebuilds And Updates

The layout rebuild path is controlled by state.gi.layout_dirty. A rebuild resets the octree, cells, probes, radiance buffers, atlas assignments, and isolated probe selection, then marks GI as updating so captures restart on the new probe set. 

if (in_state.gi.layout_dirty)
{
    gi_scene_rebuild_layout(in_gi_scene, in_state);
}
Trigger Why It Rebuilds
Updated or deleted mesh object Scene bounds and occupied octree branches may change.
Reset or transform/storage-buffer update on a mesh Object bounds may move, changing which child nodes are occupied.
GI Octree Depth slider Changes the max subdivision depth and max-depth lattice resolution.
Live-link reset All scene objects and GPU layout resources must be recreated.

Probe capture still happens incrementally. gi_scene_update() captures up to probes_to_update_per_frame probes per frame, assigns atlas slots lazily, and updates the probes buffer when newly assigned atlas indices need to reach the GPU.

Debug UI

The Global Illumination panel now exposes the octree layout directly. It shows depth, node count, payload count, non-fallback probe count, atlas usage, scene bounds, occupied cell extent range, and maximum radial depth. 

ImGui::SliderInt("GI Octree Depth", &state.gi.octree_depth, GI_Scene::min_octree_depth, GI_Scene::max_octree_depth);
ImGui::Text("Octree: depth %d  nodes %zu  payloads %d  probes %d", ...);
ImGui::Text("Cell Extent: %.2f / %.2f  Max Radial Depth: %.2f", ...);
Reading the numbers: nodes is the sparse octree size, payloads is the number of trilinear cells attached to created nodes, probes excludes the fallback probe, and Atlas includes every probe that needs an atlas slot.
