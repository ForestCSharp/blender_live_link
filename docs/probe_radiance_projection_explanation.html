probe_radiance_projection.glsl Explained

This compute shader projects one captured probe lighting cubemap into a compact probe-radiance representation. It writes either SH9 coefficients or fitted SG9 lobes, which the lighting shader can later evaluate instead of sampling the octahedral irradiance atlas.

Input Compute Output

The shader runs after a GI probe has rendered its six-face lighting cubemap. Each dispatch processes one probe. The workgroup has 32 lanes, but only lanes 0..8 own output entries: one lane per SH coefficient or SG lobe. 

layout(local_size_x=32, local_size_y=1, local_size_z=1) in;

uint coefficient_index = gl_LocalInvocationID.x;
bool has_lobe = coefficient_index < 9u;

The 32-wide workgroup is mostly a GPU/backend requirement. Sokol's shader compiler requires the compute workgroup size to be a multiple of 32 for this target, so the shader launches 32 threads. In SH mode the extra lanes return immediately. In SG mode they stay alive until the shared-memory barriers are complete, but only lanes 0..8 initialize, fit, and store lobes.

Shared Probe Radiance Helpers

The shader includes shader_common.h and then imports the shared probe_radiance block. That block defines the SH coefficient struct, the SG lobe struct, and the math for SH and SG basis evaluation. 

#include "shader_common.h"

@cs cs

@include_block probe_radiance

SH is stored as one vec4 per coefficient. RGB stores radiance data; alpha is currently unused. 

struct ProbeRadianceCoefficient
{
    vec4 value;
};

SG now stores a full lobe per entry instead of just an amplitude. params.xyz is the fitted lobe axis, params.w is fitted sharpness, and amplitude.rgb is the lobe color. 

struct ProbeSGLobe
{
    vec4 params;
    vec4 amplitude;
};

Why the buffers are flat: SH coefficients and SG lobes are addressed as probe_index * 9 + coefficient_index. This keeps GPU storage simple and lets the lighting and debug shaders use the same indexing convention.

Sampling The Sphere

Both projection paths need a repeatable set of directions over the sphere. The shader uses a Hammersley sequence: one dimension is linear, the other uses the Van der Corput radical inverse. This gives a low-discrepancy pattern that converges more evenly than pure random sampling. 

float radical_inverse_vdc(uint bits)
{
    bits = (bits << 16u) | (bits >> 16u);
    bits = ((bits & 0x55555555u) << 1u) | ((bits & 0xAAAAAAAAu) >> 1u);
    bits = ((bits & 0x33333333u) << 2u) | ((bits & 0xCCCCCCCCu) >> 2u);
    bits = ((bits & 0x0F0F0F0Fu) << 4u) | ((bits & 0xF0F0F0F0u) >> 4u);
    bits = ((bits & 0x00FF00FFu) << 8u) | ((bits & 0xFF00FF00u) >> 8u);
    return float(bits) * 2.3283064365386963e-10;
}

vec2 hammersley(uint i, uint N)
{
    return vec2(float(i) / float(N), radical_inverse_vdc(i));
}

The 2D Hammersley point is mapped to a direction on the unit sphere. The mapping uses z = 1 - 2u.x, which covers the full range from the north pole to the south pole. 

vec3 sample_sphere(vec2 u)
{
    float z = 1.0 - 2.0 * u.x;
    float r = sqrt(max(0.0, 1.0 - z * z));
    float phi = 2.0 * M_PI * u.y;
    return vec3(r * cos(phi), r * sin(phi), z);
}

Main Dispatch Flow

Each active lane handles one coefficient or lobe for the selected probe. The offset into the flat output buffer is: 

int coefficient_offset = probe_index * 9 + int(coefficient_index);
uint num_samples = uint(max(sample_count, 1));

This means dispatching once for probe_index = 17 writes entries 153..161 in the corresponding SH coefficient or SG lobe buffer.

SH9 Projection

In SH9 mode, each active lane projects the cubemap radiance onto one real spherical-harmonic basis function. For coefficient k, it repeatedly samples a sphere direction dir, reads the cubemap radiance in that direction, multiplies by sh9_basis(k, dir), and accumulates. 

if (radiance_mode == PROBE_RADIANCE_MODE_SH9)
{
    if (!has_lobe)
    {
        return;
    }

    vec3 accumulated = vec3(0.0);
    for (uint i = 0u; i < num_samples; ++i)
    {
        vec3 dir = sample_sphere(hammersley(i, num_samples));
        vec3 radiance = texture(samplerCube(cubemap_lighting_texture, smp), dir).rgb;
        accumulated += radiance * sh9_basis(int(coefficient_index), dir);
    }

    float integral_weight = (4.0 * M_PI) / float(num_samples);
    sh9_coefficients[coefficient_offset].value = vec4(accumulated * integral_weight, 0.0);
}

The 4π / sample_count factor converts the average sample sum into an approximation of an integral over the sphere. Later, the lighting shader evaluates those coefficients with diffuse convolution factors.

What SH9 Stores

SH9 is a low-frequency representation. It is good for smooth irradiance and broad directional color changes, but it cannot represent sharp lighting details.

Each probe gets nine RGB coefficients: one constant term, three first-order directional terms, and five second-order terms.

SG9 Projection

In SG9 mode, each active lane owns one spherical-Gaussian lobe. The current implementation is no longer a fixed basis with only amplitudes fitted. It starts from nine seed axes, fits lobe axes and sharpness values in shared memory, then stores both the fitted lobe shape and its color amplitude. 

const int SG9_NUM_LOBES = 9;
const int SG9_FIT_ITERATIONS = 4;

shared vec3 shared_sg9_axes[SG9_NUM_LOBES];
shared float shared_sg9_sharpness[SG9_NUM_LOBES];

Lanes 0..8 initialize the shared lobe state from sg9_initial_axis() and SG9_SHARPNESS. All 32 lanes then synchronize before fitting begins. 

if (has_lobe)
{
    shared_sg9_axes[coefficient_index] = sg9_initial_axis(int(coefficient_index));
    shared_sg9_sharpness[coefficient_index] = SG9_SHARPNESS;
}
barrier();

During fitting, each lobe computes a soft responsibility for every sample direction. The responsibility is the lobe's diffuse SG response divided by the sum of all nine diffuse SG responses for that direction. 

float sg9_basis_sum(vec3 dir)
{
    float sum_basis = 0.0;
    for (int i = 0; i < SG9_NUM_LOBES; ++i)
    {
        sum_basis += sg_lobe_diffuse_response(shared_sg9_axes[i], shared_sg9_sharpness[i], dir);
    }
    return max(sum_basis, 0.00001);
}

Each iteration moves the lobe axis toward its responsibility-weighted sample directions. The average cosine between the old axis and those samples is converted into a sharpness estimate, then clamped to the shared SG sharpness range. 

for (int iteration = 0; iteration < SG9_FIT_ITERATIONS; ++iteration)
{
    if (has_lobe)
    {
        vec3 current_axis = shared_sg9_axes[coefficient_index];
        float current_sharpness = shared_sg9_sharpness[coefficient_index];

        vec3 accumulated_direction = vec3(0.0);
        float accumulated_weight = 0.0;
        float accumulated_cosine = 0.0;

        for (uint i = 0u; i < num_samples; ++i)
        {
            vec3 dir = sample_sphere(hammersley(i, num_samples));
            float basis = sg_lobe_diffuse_response(current_axis, current_sharpness, dir);
            float responsibility = basis / sg9_basis_sum(dir);

            accumulated_direction += dir * responsibility;
            accumulated_cosine += dot(current_axis, dir) * responsibility;
            accumulated_weight += responsibility;
        }

        if (accumulated_weight > 0.00001)
        {
            vec3 fitted_axis = normalize(accumulated_direction);
            float average_cosine = clamp(accumulated_cosine / accumulated_weight, 0.0, 0.999);
            float fitted_sharpness = clamp(
                1.0 / max(1.0 - average_cosine, 0.00001),
                SG9_MIN_SHARPNESS,
                SG9_MAX_SHARPNESS
            );

            shared_sg9_axes[coefficient_index] = fitted_axis;
            shared_sg9_sharpness[coefficient_index] = fitted_sharpness;
        }
    }

    barrier();
}

After fitting, each lobe uses the final responsibility weights to estimate its color amplitude from the cubemap. The shader stores the fitted axis and sharpness in params, and the normalized RGB amplitude in amplitude. 

if (!has_lobe)
{
    return;
}

vec3 final_axis = shared_sg9_axes[coefficient_index];
float final_sharpness = shared_sg9_sharpness[coefficient_index];
vec3 accumulated_radiance = vec3(0.0);
float accumulated_weight = 0.0;
for (uint i = 0u; i < num_samples; ++i)
{
    vec3 dir = sample_sphere(hammersley(i, num_samples));
    vec3 radiance = texture(samplerCube(cubemap_lighting_texture, smp), dir).rgb;
    float basis = sg_lobe_diffuse_response(final_axis, final_sharpness, dir);
    float responsibility = basis / sg9_basis_sum(dir);

    accumulated_radiance += radiance * responsibility;
    accumulated_weight += responsibility;
}

sg9_lobes[coefficient_offset].params = vec4(final_axis, final_sharpness);
sg9_lobes[coefficient_offset].amplitude =
    vec4(accumulated_radiance / max(accumulated_weight, 0.00001), 0.0);

What SG9 Stores

SG9 stores nine complete lobes per probe. Each lobe has a fitted axis, fitted sharpness, and RGB amplitude. The initial axes and clamp limits are shared constants, but the final lobe shape is per probe.

The fit is still intentionally compact: it uses four soft-assignment iterations over Hammersley sphere samples, then a final weighted radiance average. It is more adaptive than the older fixed SG basis without turning probe updates into a full nonlinear optimizer.

SG9 Reconstruction

Runtime lighting and probe debug visualization use the same reconstruction shape. For the requested normal, they evaluate all nine fitted lobes, blend each lobe's amplitude by its diffuse response, then divide by the response sum. This keeps the fitted SG output normalized in the same spirit as the projection pass. 

vec3 reconstructed_radiance = vec3(0.0);
float weight_sum = 0.0;
int coefficient_offset = probe_index * 9;
for (int i = 0; i < 9; ++i)
{
    vec4 lobe = sg9_lobes[coefficient_offset + i].params;
    float response = sg_lobe_diffuse_response(normalize(lobe.xyz), lobe.w, normal);
    reconstructed_radiance += sg9_lobes[coefficient_offset + i].amplitude.rgb * response;
    weight_sum += response;
}
return max(reconstructed_radiance / max(weight_sum, 0.00001), vec3(0.0));

What This Shader Does Not Do

No Visibility

Probe visibility remains in the octahedral depth/visibility atlas. Chebyshev and EVRP are handled elsewhere.

No Octahedral Radiance

The octahedral irradiance atlas path is still generated by cubemap_to_octahedral.glsl. This shader only writes the SH9 coefficient buffer and SG9 lobe buffer.

No Runtime Shading

Final surface GI is evaluated in lighting.glsl, which reads the coefficients or lobes written here.

End-To-End Data Flow

LightingCapture::render(...)
    - render probe geometry cubemap
    - render probe lighting cubemap
    - render radial visibility cubemap
    - update octahedral visibility atlas
    - if radiance mode or probe visualization needs SH9:
        dispatch probe_radiance_projection.glsl
        write 9 SH coefficients for this probe
    - if radiance mode or probe visualization needs SG9:
        dispatch probe_radiance_projection.glsl
        fit and write 9 SG lobes for this probe

lighting.glsl
    - Octahedral mode: sample octahedral irradiance atlas
    - SH9 mode: evaluate SH9 coefficient buffer
    - SG9 mode: evaluate SG9 lobe buffer
    - all modes: use octahedral visibility atlas for probe occlusion
