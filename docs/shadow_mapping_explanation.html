Depth Shadow Maps vs VSM vs EVSM4

A technical comparison of the three shadowing approaches we have touched in the renderer: classic depth comparison, Variance Shadow Maps, and 4-component Exponential Variance Shadow Maps.

Executive Summary

All three techniques start with the same basic idea: render the scene from the light's point of view, then use that light-space information later while shading the camera view. They differ in what the shadow texture stores and how lighting decides whether a receiver is blocked.

Depth Shadow Map

stores depthhard compare

Stores the closest depth seen by the light. Lighting compares receiver depth against that stored depth.

VSM

stores momentsblur-friendly

Stores mean depth and mean squared depth. Lighting estimates shadow probability using Chebyshev's inequality.

EVSM4

warped momentsfilterable

Stores positive and negative exponential depth moments to make filtered moments separate blockers better.

Shared Setup: Light-Space Rendering

Before comparing the techniques, it helps to name the shared coordinate spaces. A sun shadow pass builds an orthographic light camera and renders visible scene geometry from the sun direction. The output texture is a shadow map. During the lighting pass, a world-space receiver position is transformed by the same shadow_view_projection matrix. 

vec4 shadow_clip_position = shadow_view_projection * vec4(world_position, 1.0);
vec3 shadow_ndc = shadow_clip_position.xyz / shadow_clip_position.w;
vec2 shadow_uv = shadow_ndc.xy * 0.5 + 0.5;
shadow_uv.y = 1.0 - shadow_uv.y;

Our renderer uses inverse depth for normal rendering. For the shadow-map math, we normalize depth so it increases as the point moves away from the light: 

float normalized_shadow_depth(vec3 shadow_ndc)
{
#if USE_INVERSE_DEPTH
    return 1.0 - shadow_ndc.z;
#else
    return shadow_ndc.z * 0.5 + 0.5;
#endif
}
The important convention: after normalization, smaller depth means closer to the light, larger depth means farther from the light. Bias usually moves the receiver depth slightly toward the light.

1. Classic Depth-Based Shadow Mapping

Classic shadow mapping stores one value per texel: the nearest depth seen by the light. In lighting, the receiver is shadowed if its light-space depth is greater than the stored light-space depth, after bias.

Shadow Pass Output

// Fragment shader writes no color target in the simplest version.
// Hardware depth stores the nearest surface from the light.
void main()
{
    // Depth-only pass.
}

Lighting Comparison

float receiver_depth = normalized_shadow_depth(shadow_ndc);
float stored_depth = texture(shadow_depth_texture, shadow_uv).r;
float biased_receiver = receiver_depth - shadow_bias;

float visibility = (biased_receiver <= stored_depth) ? 1.0 : 0.0;

With a single lookup, this creates a hard binary edge. A common improvement is PCF: sample several nearby depth texels, compare each one, and average the results. 

float visibility = 0.0;
for (int y = -1; y <= 1; ++y)
{
    for (int x = -1; x <= 1; ++x)
    {
        float d = texture(shadow_depth_texture, shadow_uv + vec2(x, y) * texel_size).r;
        visibility += (biased_receiver <= d) ? 1.0 : 0.0;
    }
}
visibility /= 9.0;

Properties

StrengthWeaknessTypical Fix
Conceptually simple and precise at contact points. Hard edges and visible shadow texels unless PCF is used. PCF, cascades, stable texel snapping, slope-scaled bias.
Stores only one depth value, so memory cost is low. Not directly blur-friendly: blurring depths gives invalid comparisons. Use percentage-closer filtering, not regular texture blur.

2. Variance Shadow Maps

VSM changes the stored data. Instead of storing only one depth, it stores the first two statistical moments of depth: E[x] and E[x²]. After blurring, the texture approximates a distribution of depths inside the filter footprint.

Shadow Pass Output

float depth = normalized_light_depth;
float dx = dFdx(depth);
float dy = dFdy(depth);

float moment1 = depth;
float moment2 = depth * depth + 0.25 * (dx * dx + dy * dy) + moment_bias;

out_color = vec2(moment1, moment2);

From those two values, lighting reconstructs variance: 

float mean = moments.x;
float mean_squared = moments.y;
float variance = max(mean_squared - mean * mean, min_variance);

Chebyshev Visibility

If the receiver is in front of the average blocker depth, it is considered fully lit. Otherwise VSM estimates an upper bound for how likely the receiver is to be lit. 

float chebyshev_upper_bound(float mean, float mean_squared, float receiver_depth)
{
    if (receiver_depth <= mean)
    {
        return 1.0;
    }

    float variance = max(mean_squared - mean * mean, min_variance);
    float d = receiver_depth - mean;
    return variance / (variance + d * d);
}

Since Chebyshev gives an upper bound, it tends to overestimate light. That is why VSM commonly needs a light bleeding reduction step: 

float reduce_light_bleeding(float p_max, float amount)
{
    return clamp((p_max - amount) / (1.0 - amount), 0.0, 1.0);
}

Why VSM Is Blur-Friendly

A regular Gaussian or box blur over depth values breaks classic shadow comparison. A Gaussian blur over moments, however, still gives meaningful local estimates of mean and variance. 

// Separable Gaussian blur can be used directly on VSM/EVSM moments.
result += texture(moment_texture, uv + direction * offset * texel_size) * weight;

Properties

StrengthWeaknessTypical Fix
Supports normal texture filtering and separable blur. Can leak light when depth distribution is complex. Light-bleed reduction, min variance, EVSM.
Soft shadows are cheaper than large PCF kernels. Can become splotchy or unstable with low precision. Higher precision, moment bias, tuned variance floor.

3. EVSM4

EVSM keeps the same statistical idea as VSM but changes the depth domain. Instead of storing moments of linear depth, it stores moments of exponentially warped depth. EVSM4 stores two distributions: one for positive warp and one for negative warp.

Depth Warp

const float EVSM_POSITIVE_EXPONENT = 5.0;
const float EVSM_NEGATIVE_EXPONENT = 5.0;

vec2 evsm_warp_depth(float depth)
{
    float centered_depth = depth * 2.0 - 1.0;
    return vec2(
        exp(EVSM_POSITIVE_EXPONENT * centered_depth),
        -exp(-EVSM_NEGATIVE_EXPONENT * centered_depth)
    );
}

The positive warp improves separation on one side of the distribution. The negative warp improves separation on the other side. Using both lets lighting take the stricter result.

Shadow Pass Output

vec2 warped_depth = evsm_warp_depth(depth);
vec2 dx = dFdx(warped_depth);
vec2 dy = dFdy(warped_depth);

vec2 second_moments =
    warped_depth * warped_depth +
    0.25 * (dx * dx + dy * dy) +
    vec2(moment_bias);

// EVSM4 layout:
// R = positive warped depth
// G = positive second moment
// B = negative warped depth
// A = negative second moment
out_color = vec4(
    warped_depth.x,
    second_moments.x,
    warped_depth.y,
    second_moments.y
);

Lighting Visibility

vec4 moments = texture(shadow_moments_texture, shadow_uv);
vec2 warped_receiver = evsm_warp_depth(receiver_depth);

float positive_visibility = chebyshev_upper_bound(
    moments.x,
    moments.y,
    warped_receiver.x,
    min_variance
);

float negative_visibility = chebyshev_upper_bound(
    moments.z,
    moments.w,
    warped_receiver.y,
    min_variance
);

float visibility = min(positive_visibility, negative_visibility);
visibility = reduce_light_bleeding(visibility, light_bleed_reduction);

Why EVSM4 Can Improve VSM

Plain VSM uses a linear depth distribution, which can be too weak when blocker and receiver depths are close or when the filter footprint contains multiple surfaces. EVSM's exponential warp spreads depth values apart, making the Chebyshev test more decisive after blur.

EVSM is not magic. It trades one tuning problem for a better one. Exponents, moment bias, min variance, light bleeding reduction, precision, and normal/depth bias all matter.

Half Float Exponent Constraint

With RGBA16F, exponent choice must keep both the warped value and its squared moment finite. A large exponent such as 40.0 can overflow half floats: 

exp(40) ≈ 2.35e17
half float max ≈ 65504

exp(40) is far too large for RGBA16F.
Even exp(6)^2 ≈ 162754 can overflow a second moment.

That is why a conservative first pass uses exponents around 5.0. If we later move to RGBA32F, larger exponents become possible, but at much higher bandwidth cost and often without filterability.

Biases And Tuning Knobs

Receiver Depth Bias

Bias moves the receiver toward the light, reducing self-shadowing. 

float receiver_depth = normalized_shadow_depth(shadow_ndc);
float bias = slope_scaled_shadow_bias(normal, light_direction);
receiver_depth -= bias;

Normal Offset Bias

This moves the sampled world position along the receiver normal before projection. 

vec3 shadow_sample_position =
    world_position + normalize(world_normal) * normal_offset_bias;

Moment Bias

Moment bias increases stored variance before blur. It can tame splotches but may increase light bleeding. 

second_moment = warped_depth * warped_depth + derivative_variance + moment_bias;

Minimum Variance

This clamps variance at sample time. It prevents overconfident distributions. 

float variance = max(mean_squared - mean * mean, min_variance);

Light Bleed Reduction

This suppresses weak probabilities that would otherwise leak light through blockers. 

visibility = clamp(
    (visibility - light_bleed_reduction) / (1.0 - light_bleed_reduction),
    0.0,
    1.0
);
