sokol_gfx.h Local Changes

Reference notes for the project-specific changes in game/src/extern/sokol/sokol_gfx.h. Use this when replacing Sokol with a newer upstream version so the profiler and GPU tessellation readback support do not disappear during the vendor update.

Executive Summary

This repository does not use a completely vanilla sokol_gfx.h. The current header includes a local patch set with two main responsibilities: Metal GPU profiler hooks and a tiny buffer readback API for GPU-generated tessellation counts.

Metal Profiler Hooks

Metal onlydebug UI

Adds frame lifecycle callbacks, command buffer GPU start/end timing, semaphore wait reporting, pass timing, and manual GPU scopes for the ImGui profiler.

Counter-Sampled Scopes

MTLCounterSampleBufferstage samples

Uses Metal timestamp counters to report render-pass, compute-pass, and explicit scope events. Stage-boundary sampling is enough for pass rows on Apple Silicon.

Buffer Readback

MetalD3D11tessellation

Adds a local sg_buffer_readback handle so compute-generated counter buffers can be copied back to the CPU until an indirect draw path exists.

Public API Additions

Buffer Readback API

The patch adds an opaque readback handle beside the normal Sokol resource handles: typedef struct sg_buffer_readback { uintptr_t impl; } sg_buffer_readback;. It also adds a state enum and descriptor: 

typedef enum sg_buffer_readback_state {
    SG_BUFFER_READBACKSTATE_INITIAL,
    SG_BUFFER_READBACKSTATE_PENDING,
    SG_BUFFER_READBACKSTATE_READY,
    SG_BUFFER_READBACKSTATE_FAILED,
    SG_BUFFER_READBACKSTATE_UNSUPPORTED,
} sg_buffer_readback_state;

typedef struct sg_buffer_readback_desc {
    uint32_t _start_canary;
    size_t size;
    const char* label;
    uint32_t _end_canary;
} sg_buffer_readback_desc;

The public functions are:

  • sg_make_buffer_readback(const sg_buffer_readback_desc* desc)
  • sg_destroy_buffer_readback(sg_buffer_readback readback)
  • sg_request_buffer_readback(sg_buffer_readback readback, sg_buffer buf, size_t offset, size_t size)
  • sg_query_buffer_readback_state(sg_buffer_readback readback)
  • sg_consume_buffer_readback(sg_buffer_readback readback, void* dst, size_t size)

Metal Frame Timing API

These functions are declared near the existing Metal accessors and are only meaningful for SOKOL_METAL builds:

  • sg_mtl_set_gpu_frame_timing_callback()
  • sg_mtl_set_gpu_frame_timing_frame_index()
  • sg_mtl_set_gpu_frame_lifecycle_callback()
  • sg_mtl_set_gpu_frame_wait_callback()

The timing callback reports the app frame index plus MTLCommandBuffer.GPUStartTime and GPUEndTime. The lifecycle callback reports CREATED, COMMITTED, SCHEDULED, and COMPLETED events with command-buffer and submission ids. The wait callback brackets Sokol's in-flight-frame semaphore wait in the first Metal begin-pass of a frame.

Metal Scope Timing API

Scope timing adds sg_mtl_gpu_timing_event rows and exposes detected Metal counter sampling capabilities through sg_mtl_gpu_scope_timing_caps.

  • sg_mtl_init_gpu_scope_timing()
  • sg_mtl_shutdown_gpu_scope_timing()
  • sg_mtl_query_gpu_scope_timing_caps()
  • sg_mtl_set_gpu_scope_timing_callback()
  • sg_mtl_begin_gpu_scope_timing(const char* name)
  • sg_mtl_end_gpu_scope_timing(int scope_index)

Event rows carry a name, tree depth, parent index, lane index, event index, event type, start offset in milliseconds, elapsed milliseconds, and a validity flag. Event types are frame, render pass, compute pass, and manual scope.

Metal Profiler Details

Frame Timing

The patch tracks the app's frame index separately from Sokol's internal frame counter. The game calls sg_mtl_set_gpu_frame_timing_frame_index() from game/src/render/sokol_helpers.h at frame begin. When the first Metal pass creates a command buffer, the patch captures that frame index, assigns monotonically increasing command-buffer and submission ids, and reports lifecycle events.

Completion handlers call the frame timing callback with GPUStartTime and GPUEndTime when Metal provides valid values. They also resolve any counter-sampled scope events before signaling Sokol's in-flight-frame semaphore.

Scope Timing

sg_mtl_init_gpu_scope_timing() finds the Metal timestamp counter set, checks draw, dispatch, and stage-boundary sampling support, and creates counter sample buffers. The important compatibility behavior is that stage-boundary support alone is considered enough to initialize scope timing because current Apple Silicon systems can expose pass-level stage samples even when explicit draw/dispatch boundary samples are unavailable.

Render and compute passes are automatically represented as GPU events. Render passes attach MTLRenderPassSampleBufferAttachmentDescriptor to the pass descriptor and sample from startOfVertexSampleIndex to endOfFragmentSampleIndex. Compute passes use MTLComputePassSampleBufferAttachmentDescriptor from startOfEncoderSampleIndex to endOfEncoderSampleIndex.

Manual scopes call sampleCountersInBuffer inside the active render or compute encoder. Those scopes require draw-boundary or dispatch-boundary counter sampling. When a machine only supports stage-boundary samples, manual scope begin returns a negative result and the helper layer records CPU-only fallback markers instead.

Buffer Readback Details

The readback API is intentionally small and currently exists for game/src/render/tessellation.h. GPU tessellation writes counters in a storage buffer. The CPU then reads those counters to know how many generated vertices and indices are valid.

Backend Behavior
Metal Allocates a shared MTLBuffer as staging memory. A readback request must happen while a command buffer exists and no render or compute encoder is active. It opens a blit encoder, copies from the source buffer to staging memory, and marks the readback ready from a command-buffer completed handler.
D3D11 Allocates a staging ID3D11Buffer with CPU read access. A request uses CopySubresourceRegion. Querying the state attempts a non-blocking Map(..., D3D11_MAP_FLAG_DO_NOT_WAIT) and copies data into CPU memory when ready.
Other Backends Reports SG_BUFFER_READBACKSTATE_UNSUPPORTED, emits a single warning, and asserts if code tries to request a readback. Tessellation checks backend support before depending on it.

The readback object can be destroyed while a Metal copy is pending. In that case the patch marks destroy_requested and frees the object from the command-buffer completed handler.

Upgrade Checklist

  1. Replace the vendored Sokol files with the desired upstream version, including any associated utility headers or tools needed by the project.
  2. Reapply or manually merge the local sokol_gfx.h patch groups described above: top banner, readback API, Metal profiler API, log items, backend storage, readback implementation, Metal scope helpers, command-buffer lifecycle hooks, and public API implementations.
  3. Search for every token in the touch-point table. Each one should either exist in the updated header or have an intentional replacement with the same behavior.
  4. Rebuild Sokol through the game build script. On macOS this regenerates game/bin/libsokol.a; on Windows it regenerates game/bin/sokol.lib.
  5. Run the game and inspect the profiler UI. Metal should report GPU frame timing, lifecycle progression, pass events, and a clear status message for the available counter sampling mode.
  6. Exercise GPU-planned tessellation on a Metal or D3D11 backend. The debug UI should show readback as supported, readback age should update, and generated mesh counts should not stick at zero.
  7. If testing another backend, confirm that readback is reported as unsupported once and that GPU tessellation falls back or disables itself cleanly instead of spinning on pending readbacks.

Known Limits

  • The profiler patch is Metal-specific. D3D11 has separate timing support in sokol_helpers.h, not through these Metal callbacks.
  • Manual Metal GPU scopes need draw-boundary or dispatch-boundary counter sampling. Stage-only hardware still gets pass-level rows, while explicit scopes become CPU-only markers.
  • sg_buffer_readback is implemented only for Metal and D3D11. The API is local to this project and should not be treated as upstream Sokol behavior.
  • Readback is a temporary bridge for GPU-generated counts. If the renderer later moves to indirect draw/dispatch, this patch may become smaller or unnecessary.
